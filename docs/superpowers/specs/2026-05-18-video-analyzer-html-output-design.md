# video-analyzer HTML 输出设计

## 目标

将 video-analyzer 的报告输出从 Markdown 格式改为自包含 HTML 格式，在浏览器中直接查看，利用网页特性增强可读性和交互性。

## 设计原则

- **自包含** — 单个 HTML 文件，CSS/JS 全部内联，零外部依赖，离线可用
- **模式专属设计** — 四种模式各有独立的 HTML 布局和配色
- **网页交互** — 可勾选 checkbox（行动项）、可折叠面板（讨论/章节）、原生 `<details>` 实现
- **非死板** — Claude Code 简洁现代风格，圆角面板、微妙阴影、清晰排版层级
- **管道不变** — Step 0–3 完全不动，仅改 Step 4 和 Step 5

## 共享设计语言

四个模式共用内联 CSS 基础样式：

- **排版** — system-ui 字体栈，h1 26px、h2 18px（带左侧彩色竖线标记）、正文 14px
- **色彩编码** — 蓝=结论/行动，绿=成功/教学，紫=产品/讨论，红=待跟进，灰=背景
- **最大宽度** — 860px 居中
- **打印** — @media print 隐藏折叠箭头
- **交互** — `<details><summary>` 原生折叠、checkbox 勾选（完成态删除线+灰色）

## 四种模式 HTML 模板

### 1. 会议纪要（蓝色调 #3b82f6）

```
会议纪要标签 + 标题 + 元信息（日期/时长/人数）
核心结论 — 蓝底高亮面板 + 左边框
决策记录 — 表格（# | 决策 | 背景 | 备注）
行动项 — checkbox 列表（任务名 + 描述 + 负责人标签 + 截止日期），勾选后删除线
关键讨论 — <details> 折叠面板，每条一个
待跟进事项 — 红色调表格（事项 | 跟进人 | 下一步）
会议转录摘要 — 灰底文段
页脚 — 生成时间
```

### 2. 产品分析（紫色调 #8b5cf6）

```
产品分析标签 + 标题 + 元信息（公司/日期/定位）
产品定位 — 紫底高亮面板
产品架构概览 — HTML/CSS 分层模块卡片（非 ASCII）
功能模块 — <details> 折叠面板，每个模块含：
  - 截图缩略引用框（screenshots/ 相对路径）
  - 功能列表
用户旅程 — 步骤卡片横向排列 + 箭头连接
行业应用场景 — 表格（场景 | 用途 | 关键功能）
技术特色 — 标签 + 文段
页脚
```

### 3. 教学/培训总结（绿色调 #16a34a）

```
教学总结标签 + 标题 + 元信息（难度/时长/讲师）
课程概览 — 绿底高亮面板（目标/受众）
知识结构 — 标签云分组卡片（非 ASCII 树形图）
核心知识点 — <details> 折叠章节面板，每章含：
  - 要点列表
  - 常见问题/警告提示
方案速查表 — 绿调表格（场景 | 推荐方案 | 理由）
实操步骤 — 编号列表（如适用）
课程转录摘要 — 灰底文段
页脚
```

### 4. 自定义（灰色中性）

```
标题 + 元信息
摘要 — 灰底面板
正文 — LLM 根据用户描述自由组织，可灵活使用折叠面板、表格、卡片等
转录摘要
页脚
```

## 文件改动

仅 5 个文件：

| 文件 | 改动 |
|------|------|
| `video-analyzer/SKILL.md` | Step 4 指令改为引用 HTML 模板；Step 5 输出改为 `.html` |
| `video-analyzer/references/meeting-mode.md` | 输出模板替换为会议 HTML 模板 |
| `video-analyzer/references/product-mode.md` | 输出模板替换为产品 HTML 模板 |
| `video-analyzer/references/training-mode.md` | 输出模板替换为教学 HTML 模板 |
| `video-analyzer/references/custom-mode.md` | 输出模板替换为自定义 HTML 模板 |

每个 reference 文件中的 HTML 模板包含：
- HTML 结构骨架（标签层级）
- 内联 CSS 片段（模式专属配色）
- 共享基础 CSS 引用说明

## 截图处理

- 图片分析模式下，HTML 中截图以相对路径 `<img src="screenshots/...">` 引用
- 截图文件复制到 `screenshots/` 目录（现有逻辑不变）
- 转录模式下无截图引用

## 降级模式

当零匹配帧时（transcript-only）：
- 移除截图相关内容
- 添加黄色警告横幅："⚠️ 未能匹配到目标帧，以下报告仅基于转录内容生成"
- 追加转录文件附录章节

## 非目标

- 不生成 Markdown 输出（完全替换）
- 不持久化 checkbox 状态
- 不引入外部 CSS/JS 框架
- 不改变 Step 0–3 管道逻辑
- 不新增文件
