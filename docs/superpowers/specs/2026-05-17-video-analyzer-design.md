# video-analyzer Skill 设计文档

> 日期：2026-05-17
> 状态：已批准

## 背景

当前 `video-product-summary` skill 专精于产品演示视频分析，硬编码了产品功能模块、User Journey、行业应用等模板。需要将其重构为通用的视频内容提取 skill，支持用户从视频中提取任意类型的内容。

## 目标

将 skill 从"产品演示专用"重构为"通用视频内容提取"，保留核心视频处理流水线，通过模式机制支持不同的提取需求。

## 设计决策

### 方案选择

采用**主文件 + references 分文件**方案：
- SKILL.md 包含模式选择逻辑和共享视频处理流水线
- references/ 目录下每个模式一个文件，包含分析策略和输出模板
- 选择理由：主文件精简，各模式独立维护，避免单文件过长

### 模式设计

三种预设模式 + 自定义模式：

| 模式 | 触发关键词 | references 文件 |
|------|-----------|----------------|
| 会议纪要 | 会议、meeting、纪要、行动项 | meeting-mode.md |
| 教学/培训总结 | 教学、培训、课程、知识点 | training-mode.md |
| 产品演示分析 | 产品、功能、demo、演示 | product-mode.md |
| 自定义 | 其他 / 用户明确说"提取XXX" | custom-mode.md |

### 自定义模式

用户用自然语言描述提取目标，AI 解析为三个要素：
1. **提取目标**：要提取什么
2. **关注维度**：重点看什么
3. **输出格式偏好**：怎么呈现

如果描述太模糊，AI 主动追问。

### 输出格式

每种模式有固定的 Markdown 输出模板，定义在各自的 references 文件中。

## SKILL.md 结构

### Frontmatter

```yaml
name: video-analyzer
description: "从视频中提取用户想要的内容。支持会议纪要、教学/培训总结、产品演示分析三种预设模式，也支持自然语言描述的自定义提取。当用户提到"分析视频"、"提取视频内容"、"视频总结"、"会议纪要"、"视频转文档"时触发。即使用户没有明确说"视频"，只要上下文涉及视频文件且需要从中提取信息，也应使用此 skill。"
```

### Step 0：模式识别

根据用户输入关键词匹配模式，读取对应 references 文件。

### Step 1-2.5：共享视频处理流水线

与当前 skill 基本一致，核心改动：
- Step 2.5b 泛化为"识别目标内容时间段 vs 无关段"
- 过滤标准根据模式动态决定：
  - 会议纪要：关注有实质性讨论的段落，跳过寒暄/闲聊
  - 教学总结：关注知识讲解段落，跳过课间休息/广告
  - 产品分析：关注功能演示段落（保持现有逻辑）
  - 自定义：根据用户描述动态生成过滤标准
- 帧过滤边界容忍度（±3 秒）和降级模式（零目标帧 → 纯转录模式）保持不变

### Step 3：分析执行

读取对应模式的 references 文件，按其中的分析策略执行。每个 references 文件定义：
- 帧分析 prompt（传给 MiniMax understand_image）
- 转录提炼重点
- 降级模式（纯转录）的分析策略

### Step 4-6：输出、附录、标注

共享逻辑，但使用各模式的输出模板。

## 各模式输出模板概要

### 会议纪要模式

1. 会议概要（时间、参与者、主题）
2. 决策记录
3. 行动项清单（表格：任务 | 负责人 | 截止时间 | 状态）
4. 关键讨论点
5. 待定/未决事项
6. 会议转录摘要
7. 附录

### 教学/培训总结模式

1. 课程概览（主题、讲师、时长）
2. 知识结构（章节树）
3. 核心知识点详解
4. 关键概念速查表
5. 实操步骤（如有）
6. 课程转录摘要
7. 附录

### 产品演示分析模式

基本沿用当前 skill 的输出模板（功能模块详解、User Journey、架构图等），从当前 SKILL.md 迁移。

### 自定义模式

通用骨架：标题 + 概要 + 正文（AI 根据提取目标组织） + 附录。

## 不变的部分

- 前置依赖：peepshow、MiniMax MCP、ffmpeg、whisper-cli
- 视频处理流水线：ffprobe → peepshow → 转录 → 帧过滤
- 帧分析方式：分两轮（采样摸底 + 查漏补缺），使用 MiniMax understand_image
- 降级模式：零目标帧时进入纯转录模式
- 并行调用、减帧策略、中文输出、隐私安全等注意事项
